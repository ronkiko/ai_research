# Иерархия учебных задач для нейросетей

Краткий справочник: какие задачи решаются простыми моделями, а какие требуют
более сложных архитектур. Используется в `game1` для выбора столов под
конкретную модель.

## Уровень 0 — линейно разделимые

Задачи, где один нейрон (или линейный классификатор) справляется без скрытых
слоёв.

| Пример | Что нужно | Модель |
|---|---|---|
| «Куда покатится мяч» (`ball`) | Предсказать перекос влево/вправо | `bias` — одно число-привычка |
| «Хитрый дилер» без контекста | Инвертировать один вход | `logistic` — один вес + смещение |
| Линейная регрессия | `y = kx + b` | один нейрон |
| AND/OR с правильными весами | линейно разделимые логики | один нейрон |

**Почему хватает:** граница между классами — прямая линия (или гиперплоскость).

## Уровень 1 — нелинейно разделимые, один скрытый слой

Задачи, где нужно объединить несколько линейных границ. Один скрытый слой с
нелинейной активацией теоретически аппроксимирует любую функцию, но на практике
сложные паттерны требуют слишком много нейронов.

| Пример | Что нужно | Модель |
|---|---|---|
| «Детектор лжи» (`witness`) | XNOR: совпадают ли два входа | `mlp` / `torch`, 2→4→1 |
| XOR классический | входы разные → 1 | `mlp` / `torch`, 2→4→1 |
| Круг/кольцо в 2D | область внутри окружности | MLP 2→N→1 |
| Простая функция | `y = sin(x)` на отрезке | MLP с Tanh |

**Почему хватает:** функцию можно разрезать на кусочки и собрать из них.

## Уровень 2 — память / последовательности

Задачи, где ответ зависит не только от текущего входа, но и от предыдущих.
MLP с фиксированным окном справляется только для ограниченной глубины памяти.

| Пример | Что нужно | Модель |
|---|---|---|
| «Кормушка» (`kormushka`) | Учесть прошлую сторону | `logistic` — один шаг памяти |
| «Скрытый паттерн» (`pattern`) | Предсказать следующий бит по 3 прошлым | MLP с окном |
| Чётность длинной последовательности | XOR всех битов | LSTM / Transformer |
| Длинный паттерн (N=8–100) | Помнить далёкое прошлое | LSTM / Transformer |
| Палиндром переменной длины | Сравнить начало и конец | LSTM / Transformer |
| Языковое моделирование | Предсказать следующее слово | Transformer |

**Почему MLP не справляется:** окно фиксированного размера не масштабируется,
а веса не разделяются во времени.

## Уровень 3 — пространственная структура

Задачи, где вход имеет сетчатую/пространственную структуру: соседние элементы
важнее далёких.

| Пример | Что нужно | Модель |
|---|---|---|
| MNIST-lite (4×4 / 8×8) | Классифицировать цифру | CNN |
| MNIST (28×28) | Распознать рукописную цифру | CNN |
| CIFAR-10 | Классифицировать цветное изображение | CNN + пулинг |
| Детекция объектов | Найти объект на картинке | CNN + YOLO / R-CNN |

**Почему CNN:** свёртки улавливают локальные признаки (края, углы) и
разделяют параметры по всему изображению.

## Уровень 4 — пространство + время

Задачи, где есть и пространственная, и временная структура.

| Пример | Что нужно | Модель |
|---|---|---|
| Видео | Классифицировать действие в видеоряде | CNN + LSTM/Transformer |
| Аудио | Распознать речь | CNN + RNN / Transformer |
| Atari-игры | Играть по пикселям экрана | CNN + DQN / policy-gradient |
| «Плывущий мяч» (`drift`) с картинкой ветра | Адаптироваться к визуальным сигналам | CNN + адаптивная головка |

## Уровень 5 — генерация / планирование / рассуждение

Задачи, где нужно не просто классифицировать или предсказать, а сгенерировать
сложную структуру или найти оптимальную стратегию.

| Пример | Что нужно | Модель |
|---|---|---|
| Генерация текста | Продолжить предложение | Transformer (GPT) |
| Генерация изображений | Создать картинку по описанию | GAN / diffusion |
| Шахматы / Go | Найти лучший ход в глубоком дереве | MCTS + Transformer / ResNet |
| Кодирование задач | Решить алгоритмическую задачу | большой Transformer + RL |

## Как это используется в game1

Каждый стол в `game1` привязан к уровню сложности. Физически столы лежат в
подпапках `modules/core/mechanics/levelN/`:

```
modules/core/mechanics/
├── level0/          # линейно разделимые
│   ├── ball.py
│   └── dealer.py
├── level1/          # один скрытый слой
│   ├── drift.py
│   ├── kormushka.py
│   └── witness.py
└── level2/          # память / последовательности
    └── pattern.py
```

При выборе стола и модели стоит ориентироваться на таблицу:

| Стол | Уровень | Минимальная модель | Путь |
|---|---|---|---|
| `ball` | 0 | `bias` | `mechanics/level0/ball.py` |
| `dealer` (инверсия) | 0–1 | `logistic` / `mlp` | `mechanics/level0/dealer.py` |
| `kormushka` | 0–1 | `logistic` / `mlp` | `mechanics/level1/kormushka.py` |
| `witness` | 1 | `mlp` / `torch` со скрытым слоем | `mechanics/level1/witness.py` |
| `pattern` | 1–2 | MLP с окном / LSTM | `mechanics/level2/pattern.py` |
| `drift` | 1 | `bias` / `mlp` с адаптацией | `mechanics/level1/drift.py` |

## Кандидаты на следующие столы

Чтобы продвинуть game1 дальше, можно добавить:

1. **«Чётность»** — последовательность битов, ответ = чётность (XOR всех).
   - Уровень 2. MLP с окном упрётся на длинных последовательностях, нужен LSTM.
2. **«Длинный паттерн»** — паттерн длины 8–16, агент видит только 2 последних
   исхода.
   - Уровень 2. Требует долгосрочной памяти.
3. **«Палиндром»** — последовательность длины N, ответ 1 если симметрична.
   - Уровень 2. Требует память о далёких позициях.
4. **«MNIST-lite»** — бинарное изображение 8×8, классификация цифр.
   - Уровень 3. Нужна CNN.

При добавлении нового стола:
1. Физически положить файл в `modules/core/mechanics/levelN/<key>.py`, где N —
   уровень сложности по таблице выше. Если уровня ещё нет — создать папку и
   добавить уровень в древовидный браузер (`engine.py`, список `LEVEL_FOLDERS`).
2. Добавить импорт класса в `modules/core/casino.py` из нового пути.
3. Добавить строку в `_TABLES` в `casino.py`.
4. Обновить:
   - `RULES` и альтернативную постановку в файле стола;
   - таблицу задач в `README.md`;
   - таблицу путей и уровней в этом файле (`RESEARCH_TASKS.md`);
   - древовидный браузер в `engine.py` — добавить группу/уровень и сам стол.
