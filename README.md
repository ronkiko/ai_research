# ai_research

Зонтичный исследовательский проект по изучению подходов к обучению ИИ-моделей
действовать и вести себя **более по-человечески**.

## Суть

Цель — не игры как продукты, а **игры как управляемые экспериментальные среды**.
Каждая игра — это маленькая сконструированная сцена (стресс, выбор, ненадёжный
собеседник, ресурсы, эмоции), где ИИ-агент вынужден вести себя «как живой
персонаж в беде/под давлением», а логи партий становятся материалом для изучения:
что именно делает поведение модели более живым и последовательным, и как это
прививать (в том числе через реальное изменение весов локальной сети).

Проект вырос из игры `~/hunter2.py` (Hunter) — ролевой ловушки для Кими через
Ollama. Hunter оказался слишком сложным для чистого исследования и быстро
оброс костылями, поэтому подход перерождается здесь в виде **простых,
независимых игр-экспериментов**, каждая со своей гипотезой.

## Структура

```
ai_research/
├── README.md         ← этот файл: общая идея и стек
├── AGENTS.md         ← короткие правила для агентов
├── CLAUDE.md         ← короткие правила для Claude
└── gameN/            ← отдельное исследование
    ├── README.md     ← детально: правила, гипотеза, что меряем
    └── AGENTS.md     ← (при необходимости) локальные правила агентов
```

Каждая подпапка `gameN/` — **независимое исследование** на общем стеке:

- `game1/` — первая игра (правила и гипотеза — в её README).
- `game2/`, `game3/`, … — будущие эксперименты, отдельные задачи/вопросы.

Подпапки самостоятельны: можно работать с одной, не трогая остальные.

## Общий стек

- **Python 3.10+**
- **PyTorch 2.1.2 (CPU)** — локальная обучаемая нейросеть (веса реально меняются).
  Другие установленные библиотеки: numpy, scipy, scikit-learn, pandas, onnxruntime.
- **Адаптер ИИ** — единый интерфейс «наблюдение → действие/реплика» (+ фаза
  обучения `train()`), за которым скрывается конкретный бэкенд.
  Сейчас разрабатывается **только локальный обучаемый бэкенд**. Подключение Kimi
  через Ollama оставлено на будущее — интерфейс держим расширяемым, но не
  реализуем сейчас.

## Как работать с проектом

1. Прочитай **этот** README — общая идея и стек.
2. Перед работой с конкретной игрой читай **её** `gameN/README.md` (и при наличии
   `gameN/AGENTS.md`) — там правила, гипотеза и локальные ограничения.
   **При работе внутри подпапки приоритет у её ридми/агент-файлов**, а не у
   корневых.
3. Корневые `AGENTS.md` / `CLAUDE.md` — только краткие указатели, не дублируют
   детали.
