# ai_research

Зонтичный исследовательский проект: игры как управляемые среды для изучения
человекоподобного поведения ИИ-агентов.

Цель — не игры как продукты, а чистые эксперименты. Каждая игра — маленькая
сконструированная сцена (выбор, ненадёжный мир, дефицит информации), где модель
вынуждена вести себя «как живой персонаж в условиях неопределённости».
Логи партий — материал для анализа: что именно делает поведение модели более
живым и как это прививать через изменение весов локальной сети.

## Структура

```
ai_research/
├── README.md          ← этот файл: общая идея и стек
├── AGENTS.md          ← краткие правила для агентов
├── CLAUDE.md          ← краткие правила для Claude
├── .gitignore
└── gameN/             ← отдельное исследование
    ├── README.md      ← детально: правила среды, гипотеза, что и как меряем
    └── AGENTS.md      ← (опционально) локальные правила разработки
```

Каждая подпапка `gameN/` — **независимое исследование** на общем стеке.
Подпапки самостоятельны: можно работать с одной, не трогая остальные.

- [`game1/`](game1/) — первая игра (curses TUI): азартные мини-игры, обучаемые
  модели, субменю. Подробности — в её `README.md`.

## Стек

| Компонент | Детали |
|-----------|--------|
| Python 3.10+ | |
| PyTorch 2.1+ (CPU) | локальная обучаемая нейросеть (веса реально меняются) |
| numpy / scipy / scikit-learn / pandas / onnxruntime | окружение |
| curses | TUI (минимальный терминал 80×24) |

Сейчас реализуется **только локальный обучаемый бэкенд**. Ollama/Kimi оставлен
на будущее — интерфейс держим расширяемым, но не реализуем сейчас.

## Как работать

1. Прочитай этот README — общая идея и стек.
2. Перед работой с конкретной игрой читай **её** `gameN/README.md` (и при
   наличии `gameN/AGENTS.md`) — там правила среды, гипотеза и локальные
   соглашения.
3. **При работе внутри подпапки приоритет у её локальных файлов**, не у корневых.
   Решения одной игры не переноси в другую.
4. Корневые `AGENTS.md`/`CLAUDE.md` — краткие указатели для агентов, не
   дублируют детали.
